8 How to create embedding samples
9 ===============================
10 
11 Checkout the code using the following command:
12 
13  git clone --recursive https://github.com/star-bnl/star-vertex-eval.git
14 
15 Select desired zerobias files from the database:
16 
17  get_file_list.pl -keys path,filename,events -cond 'filename~st_zerobias_adc_1412%,year=2013,runnumber>14125000' -delim / -limit 20
18 
19 Create a text file with a list of zerobias data files in the following format
20 (`path/to/file` `run_id` `num_of_events`), e.g.:
21 
22  $cat supple/filelist_zerobias.txt
23  /star/data05/daq/2013/embedding/st_zerobias_adc_14150005_raw_3330002.daq 14150005 103
24  /star/data05/daq/2013/embedding/st_zerobias_adc_14150004_raw_1530002.daq 14150004 103
25  ...
26  /star/data05/daq/2013/embedding/st_zerobias_adc_14150005_raw_8660002.daq 14150005 102
27  ...

40 J/psi
41 -----
42 
43 Create a directory from where condor jobs will be submitted and run the script
44 as:
45 
46  mkdir submit_jpsi && cd submit_jpsi
47  /path/to/star-vertex-eval/supple/submit_jobs_embedding_jpsi.sh 200
48 
49 If the package was not checked out in the default location of
50 ${HOME}/star-vertex-eval/ then you should specify the correct location as:
51 
52  SOURCE_DIR=/path/to/star-vertex-eval/ \
53  OUTPUT_DIR=/tmp/jpsi_embed \
54  /path/to/star-vertex-eval/supple/submit_jobs_embedding_jpsi.sh 200
55 
56 There is a number of other script parameters which can be modified if the
57 default values are not appropriate. Search for "NAMED ARGUMENTS" to find these
58 values.

61 Study the ranking scheme in STAR
62 ================================
63 
64 This section is outdated but left here for the reference until after all
65 mentioned macros are removed from the repository...
66 
67 1.- The macro VertexRank.C will read the MuDst sample and create a tree with some vertex information. To run (e.g. 100 events and output file outvertex.root):
68 
69  $root -l -b -q lMuDst.C 'VertexRank.C(100, "path/to/*.MuDst.root", "outvertex")'
70 
71 2.- Run TMVA to classify signal and background. A separate set of signal and background must be given, with a define set of variables for the training. Also need to specify the classifier. To run (e.g. with classifiers Fisher and Likelihood):
72 
73  $root TMVAClassification.C\(\"Fisher,Likelihood\"\)
74 
75 3.- Apply the results of the classification with ApplyClassifier.C. The output will be the same tree for step (1) but with a new branch with the classifier response. Run:
76 
77  $root ApplyClassifier.C

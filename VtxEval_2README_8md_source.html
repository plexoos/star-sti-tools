1 VtxEval: Vertex Reconstruction Evaluation and Efficiency at STAR
2 ================================================================
3 
4 To build the library and the `vtxeval` executable follow the instructions in the
5 project's top level [README.md](../README.md). Since in this study we are
6 focusing on evaluation of the vertex finders one can build and install only the
7 relevant libraries:
8 
9  $ mkdir build/ && cd build/
10  $ cmake -D CMAKE_INSTALL_PREFIX=../build-install \
11  -D BOOST_ROOT=$OPTSTAR \
12  -D CMAKE_BUILD_TYPE=Release ../
13  $ make -j4 star-vertex
14  $ make install

17 How to create embedding samples
18 ===============================
19 
20 Use STAR's `get_file_list.pl` to select a set of desired zerobias files from the
21 database. Here is an example command with some selection constraints:
22 
23  $ get_file_list.pl -keys path,filename,events \
24  -cond filename~st_zerobias_%,runnumber[]14115007-14120000,sanity=1,filetype=online_daq \
25  -delim / -limit 0
26 
27 You may choose to copy the files from `hpss` to some location on disk. In that
28 case, the paths to files returned by `get_file_list.pl` can be adjusted to point
29 to the actual file location. The final list of zerobias data files has to be in
30 the following format:
31 
32  string:"path/to/file.daq run_id num_of_events_to_process"
33 
34 The contents of your file may look like the following:
35 
36  $ cat filelist_zerobias.lis
37  ...
38  string:"/star/data05/daq/2013/embedding/st_zerobias_adc_14115007_raw_1540001.daq 14115007 123"
39  string:"/star/data05/daq/2013/embedding/st_zerobias_adc_14115007_raw_2440001.daq 14115007 155"
40  ...
41  string:"/star/data05/daq/2013/embedding/st_zerobias_adc_14115007_raw_3330001.daq 14115007 111"
42  ...

45 W boson
46 -------
47 
48 The script to submit W embedding jobs to condor is called
49 `submit_jobs_embedding_wbos.sh` and can be found in `star-travex/supple/`. There
50 is a number of named parameters which user may need to set to appropriate
51 values. The two parameters likely to be changed often are `OUT_DIR` and
52 `INPUT_FILE_LIST`. All named variables can be passed to the script in command
53 line like this:
54 
55  $ OUT_DIR=/tmp/scratch/wbos_embed_results/ \
56  INPUT_FILE_LIST=~/star-travex/supple/filelist_zerobias.lis \
57  ~/star-travex/supple/submit_jobs_embedding_wbos.sh

71 J/psi
72 -----
73 
74 Create a directory from where condor jobs will be submitted and run the script
75 as:
76 
77  mkdir submit_jpsi && cd submit_jpsi
78  /path/to/star-travex/supple/submit_jobs_embedding_jpsi.sh 200
79 
80 If the package was not checked out in the default location of
81 ${HOME}/star-travex/ then you should specify the correct location as:
82 
83  SOURCE_DIR=/path/to/star-travex/ \
84  OUTPUT_DIR=/tmp/jpsi_embed \
85  /path/to/star-travex/supple/submit_jobs_embedding_jpsi.sh 200
86 
87 There is a number of other script parameters which can be modified if the
88 default values are not appropriate. Search for "NAMED ARGUMENTS" to find these
89 values.

93 Study the ranking scheme in STAR (deprecated)
94 =============================================
95 
96 This section is outdated but left here for the reference until after all
97 mentioned macros are removed from the repository...
98 
99 1.- The macro VertexRank.C will read the MuDst sample and create a tree with some vertex information. To run (e.g. 100 events and output file outvertex.root):
100 
101  $root -l -b -q lMuDst.C 'VertexRank.C(100, "path/to/*.MuDst.root", "outvertex")'
102 
103 2.- Run TMVA to classify signal and background. A separate set of signal and background must be given, with a define set of variables for the training. Also need to specify the classifier. To run (e.g. with classifiers Fisher and Likelihood):
104 
105  $root TMVAClassification.C\(\"Fisher,Likelihood\"\)
106 
107 3.- Apply the results of the classification with ApplyClassifier.C. The output will be the same tree for step (1) but with a new branch with the classifier response. Run:
108 
109  $root ApplyClassifier.C

1 StiScan: Detector Material Evaluation at STAR
2 =============================================
3 
4 How to use
5 ==========
6 
7 To build the library and the executable follow the installation instructions in
8 the project's [README.md](../README.md). Once the `StiScan` library is built it
9 can be loaded and the `StiScanTreeMaker` can be added to the reconstruction
10 chain by using the `StiScanTreeMaker` option as:
11 
12  root4star -b -q -l 'bfc.C(<first_event>, <last_event>, "<your_bfc_options> StiScanTreeMaker", "<input_file>")'
13 
14 For a concrete example see the section below. The `bfc.C` macro can take either
15 real or simulated data as input but for our studies of the detector material
16 distribution we focus on specially simulated event samples with tracks
17 perpendicular to the colliding beams. Such configuration of tracks makes the
18 interpretation of the final results much easier. For reference, we include
19 a simple `starsim` kumac (`supple/starsim_zslice_single_z.kumac`) to produce
20 events with tracks in transverse planes at arbitrary integer `z` values. To use
21 this kumac from your build directory just do:
22 
23  export STISCAN_Z=7
24  starsim -w 0 -b ../supple/starsim_zslice_single_z.kumac
25 
26 We also include a number of support shell scripts to automate the generation of
27 such events for different `z` values by running condor jobs on the farm (see
28 below).
29 
30 Another way to produce a file with tracks generated in `z=const` planes is to
31 run a special ROOT macro (`supple/starsim_stepper.C`):
32 
33  root4star -b -l -q '../supple/starsim_stepper.C(7,10,50)'
34 
35 which in addition to a usual `starsim_zslice.fz` file will produce a ROOT file
36 (`starsim_zslice.track_history.root`) with a TTree containing the information
37 about the track as GEANT propagates it through the detector material. That file
38 can be used together with the `*.fz` one in `stiscan` in order to produce
39 comparisons between the geometries as seen by the simulated and reconstructed
40 tracks.
41 
42 The `starsim` commands from above examples should produce a `starsim_zslise.fz`
43 file that can be processed with the `bfc.C` macro as indicated earlier. The
44 `StiScanTreeMaker` option creates a ROOT file (\*.stiscan.root) with a TTree
45 that can be used to study track losses in the detector volumes with the help of
46 `stiscan` program. `stiscan` accepts either a ROOT file with an stiscan TTree
47 or a text file with a list of such ROOT files (one per line) as input. To
48 produce a set of basic histograms with track energy lost in Sti volumes one can
49 do:
50 
51  stiscan -f path/to/my.stiscan.root
52 
53 For more options type `stiscan --help`.
54 
55 By default, only sensitive layers of the PXL, IST, and SSD detectors are used in
56 the energy loss analysis. One can easily specify any other volume or a set of
57 volumes to be considered in the analysis by using regular PERL expressions. Here
58 are a few examples:
59 
60  stiscan -f path/to/my.stiscan.root -p "^.*IDSM_1/PXMO_1/PXLA_[\d]+/PXT[RML]_.*$" -g
61  stiscan -f path/to/my_stiscan_list -l path/to/my_volume_name_pattern_list -s 0.10 -g
62 
63 Either a regex pattern or a text file with a list of regex patterns
64 (`my_volume_name_pattern_list` in the above example) can be provided. It is used
65 to match the names of Sti/TGeo volumes and to select only these specific physical
66 volumes.
67 
68 *Note:* The program expects to find a ROOT file (`y2014a.root`) with the full
69 STAR geometry in the directory from where it was started. Such file can be
70 produced with the following command:
71 
72  root -l '$STAR/StarVMC/Geometry/macros/viewStarGeometry.C("y2014a")'

75 How to produce and reconstruct massive simulation
76 =================================================
77 
78 A couple of shell scripts are provided to simplify the submission of condor jobs
79 to the farm. The following commands can be used as examples:
80 
81  supple/submit_jobs_starsim_zslice.sh supple/job_template_starsim_zslice_kumac.xml
82  supple/submit_jobs_starsim_zslice.sh supple/job_template_starsim_zslice_macro.xml
83 
84  supple/submit_jobs_stiscan.sh /path/to/filelist_fz
85 
86 `submit_jobs_stiscan_zslice.sh` takes an xml template as the only parameter. In
87 our case the `starsim` command is executed to produce \*.fz files with simulated
88 events.
89 
90 `submit_jobs_stiscan.sh` utilizes the `supple/job_template_stiscan.xml`
91 template to run over the \*.fz files specified in the input list. The jobs run
92 the standard reconstruction and in addition creates a ROOT file with an stiscan
93 tree. The latter can be controlled by changing the BFC options hard-coded in the
94 script.

97 Reconstruction chain examples
98 =============================
99 
100 Run over an fzd file produced by `starsim` with the following options to produce
101 a MuDst.root file with simulated tracker response and reconstruced events.
102 
103  root4star -q -b -l 'bfc.C(1, 100, "fzin y2015a AgML usexgeom FieldOn tpcRS MakeEvent VFMinuit Sti BAna Idst NoSsdIt NoSvtIt StiHftC TpcHitMover TpxClu l0 Tree logger tpcDB tags emcY2 EEfs -dstout IdTruth big MiniMcMk clearmem StiScanTreeMaker", "starsim_zslice.fz")'

106 Regular expressions matching geometry volumes used at STAR
107 ==========================================================
108 
109  Beam pipe ^.*IDSM_1/PIPI_1/PBES_1$
110  Field cages ^.*TPCE_1/T[IO]FC.*$
111  PXL all volumes ^.*IDSM_1/PXMO_.*$
112  PXL sensitive volumes ^.*IDSM_1/PXMO_1/PXLA_[\d]+/LADR_\d/PXSI_[\d]+/PLAC_1.*$
113  PXL ribs ^.*IDSM_1/PXMO_1/PXLA_[\d]+/PXRB.*$
114  ^.*IDSM_1/PXMO_1/PXLA_[\d]+/PXLB.*$
115  IST all volumes ^.*IDSM_1/IBMO_.*$
116  IST sensitive volumes ^.*IDSM_1/IBMO_1/IBAM_[\d]+/IBLM_\d/IBSS_1.*$
117  SST all volumes ^.*IDSM_1/SFMO_.*$
118  SST sensitive volumes ^.*IDSM_1/SFMO_1/SFLM_[\d]+/SFSW_[\d]+/SFSL_1/SFSD_1.*$

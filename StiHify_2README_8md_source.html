1 StiHify: Track Reconstruction Evaluation at STAR
2 ================================================
3 
4 Introduction
5 ============
6 
7 The efficiency to find a hit in an active detector volume can be described
8 using the following components:
9 
10  P_measured = (P_instrumental + P_background) * P_corr * P_tracking
11 
12 where
13 
14 - `P_instrumental` (aka `P_detector`) is the probability for the
15 detector active media to produce a signal that can be recorded (Includes
16 efficiency of hardware and/or low level thresholds, pedestals, etc)
17 
18 - `P_background` is the probability to have a hit within the window that is not
19 produced by the considered track (for example detector noise, other tracks, etc)
20 
21 - `P_corr` include cluster/hit reconstruction efficiency, uncorrected unknown
22 distortions, inefficiency of known corrected distortions
23 
24 - `P_tracking` is the pure tracking efficiency, i.e. the probability to assign
25 a hit to the track by the tracking algo. Includes algo specific requirements
26 such as number of time hits can be used/reused, hit acceptance/rejection cuts
27 (chi^2), etc.

30 How to use
31 ==========
32 
33 To build the library and the executable follow the installation instructions in
34 the project's [README.md](../README.md). Once the `StiHify` library is built it
35 can be loaded and the `StiHifyTreeMaker` can be added to the reconstruction
36 chain by using the `StiHifyTreeMaker` option as:
37 
38  root4star -b -q -l 'bfc.C(<first_event>, <last_event>, "<your_bfc_options> StiHifyTreeMaker", "<input_file>")'
39 
40 In order to disable a sensitive Sti volume (with hits) one can provide a file
41 called `deactivate_sti_detectors.txt` with a list of regex patterns matching the
42 name of the volume to skip in tracking. Providing this file is optional but
43 required if one would like to estimate the true single hit efficiency of
44 a detector.
45 
46 By default, all tracks and track nodes will be saved in the output tree. This is
47 an unlikely desirable feature as it can lead to a very large output tree. To
48 limit the number of saved track and track nodes one can provide another text
49 file called `save_sti_detectors.txt` containing regex patterns matching the name
50 of the Sti volumes crossed by tracks. In the simplest case both the
51 `deactivate_sti_detectors.txt` and `save_sti_detectors.txt` files can contain
52 the same set of regex patterns.
53 
54 Once the tree is produced one can run the `stihify` program over it to create
55 a set of histograms with hit efficiencies.

58 Reconstruction chain examples
59 =============================
60 
61 Note that in the following example the PXL hits are excluded from tracking
62 by using the `NoPxlIT`option.
63 
64  root4star -q -b -l 'bfc.C(1, 100, "P2014a mtd btof pxlHit istHit NoPxlIT BEmcChkStat CorrX OSpaceZ2 OGridLeak3D -hitfilt StiHifyTreeMaker", "st_physics_file.daq")'
65 
66 The following command is for the pretty much standard reconstruction with all
67 HFT hits used in tracking.
68 
69  root4star -q -b -l 'bfc.C(1, 100, "P2014a mtd btof pxlHit istHit sstHit BEmcChkStat CorrX OSpaceZ2 OGridLeak3D -hitfilt StiHifyTreeMaker", "st_physics_file.daq")'

72 Regular expressions matching geometry volumes used at STAR
73 ==========================================================
74 
75  Beam pipe ^.*IDSM_1/PIPI_1/PBES_1$
76  Field cages ^.*TPCE_1/T[IO]FC.*$
77  PXL all volumes ^.*IDSM_1/PXMO_.*$
78  PXL sensitive volumes ^.*IDSM_1/PXMO_1/PXLA_[\d]+/LADR_\d/PXSI_[\d]+/PLAC_1.*$
79  PXL ribs ^.*IDSM_1/PXMO_1/PXLA_[\d]+/PXRB.*$
80  ^.*IDSM_1/PXMO_1/PXLA_[\d]+/PXLB.*$
81  IST all volumes ^.*IDSM_1/IBMO_.*$
82  IST sensitive volumes ^.*IDSM_1/IBMO_1/IBAM_[\d]+/IBLM_\d/IBSS_1.*$
83  SST all volumes ^.*IDSM_1/SFMO_.*$
84  SST sensitive volumes ^.*IDSM_1/SFMO_1/SFLM_[\d]+/SFSW_[\d]+/SFSL_1/SFSD_1.*$
